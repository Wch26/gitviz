# 申请文案 — GitViz 项目

## 问题 04：使用 Agent 或 AI 驱动构建的具体成果

### 项目名称
**GitViz — 多智能体协作的 Git 仓库可视化管理系统**

### 核心痛点

团队日常使用 Git 面临三个普遍问题：

1. **命令行反馈不直观** — `git log --oneline --graph` 输出的 ASCII 拓扑图难以阅读，新人常因记错命令导致误操作（硬回退误删代码、分支切换混乱）
2. **版本回退缺乏安全机制** — 命令行回退没有可视化确认，hard reset 不可逆，缺乏"操作前预览"和"自动备份"
3. **历史追溯低效** — Code Review 时无法快速穿梭到过去的某个提交、下载其快照进行对比验证，需要手动执行多步 git 命令

### 核心逻辑流（多 Agent 协作 + 长链推理）

本项目由 **Orchestrator + 多个 Specialist Agent** 协作构建，体现了从需求分析到功能交付的完整长链推理过程：

**第一阶段：架构设计与技术选型**
- Agent 分析需求后确定 Flask + 原生 JavaScript SPA 架构
- 拆解为 5 个模块：Git 操作封装层、RESTful API 路由、深色主题前端 UI、测试数据生成器、申请文档

**第二阶段：Git 操作层构建（git_ops.py）**
- Agent 封装 20+ 个 Git 子命令，使用 subprocess 而非 GitPython（零额外依赖）
- 处理跨平台兼容：Windows GBK 编码问题、CRLF 行尾、路径反斜杠转义
- 自建异常体系，所有操作带超时机制和兜底返回值

**第三阶段：后端 API 构建（app.py）**
- Agent 设计 17 个 RESTful 端点
- 全局 JSON 异常处理器，避免前端收到 HTML 错误页
- 实现最近仓库持久化、文件上传/下载流式传输、回退历史备份

**第四阶段：前端 UI 构建（index.html）**
- Agent 独立构建 1000+ 行单页应用，含 500+ 行 CSS（GitHub 暗色主题）
- 核心组件：提交时间线、Diff 查看器、分支管理器、回退确认对话框、快速扫描面板
- **长链推理体现**："打开仓库 → 并行加载日志/分支/状态 → 用户点击提交 → 解析 Diff → 渲染代码差异 → 选择回退模式 → 备份 ZIP → 执行回退 → 刷新所有视图"

**第五阶段：测试数据与迭代优化**
- 创建包含 5 个分支、11 次提交、1 个 tag 的测试仓库
- 经过多轮测试修复：编码解析崩溃（GBK→UTF-8）、HTML onclick 路径转义、`subprocess.stdout` 空指针保护

### 关键功能

| 功能 | 实现要点 |
|------|---------|
| 仓库自动发现 | 扫描本地目录，自动检测 `.git` 文件夹 |
| 提交历史可视化 | 时间线展示 + 分支标签 + 合并标记 |
| HEAD 位置标记 | 脉冲动画蓝色边框 + "📍 当前位置" 标识 |
| 版本回退 | soft / mixed / hard 三种模式，带确认和自动备份 |
| 快照下载 | 任意提交一键打包为 ZIP，走浏览器原生下载 |
| Diff 查看 | 增删行分别绿色/红色高亮 |
| 分支管理 | 查看/切换分支，当前分支标记 |
| 工作区状态 | 可视化 git status |

### 技术指标
- 代码总量：1500+ 行（后端 300+、前端 1200+、测试脚本 150+）
- API 端点：17 个 RESTful
- Git 操作：完整支持 log / diff / archive / reset / checkout / branch / stash / status
- 零外部前端依赖
- 跨平台兼容（Windows / macOS / Linux）

## 快速运行指南

```
cd GitViz                         # 进入项目目录
pip install flask                 # 安装依赖（仅 Flask）
bash setup_demo_repo.sh           # 创建测试仓库
python app.py                     # 启动服务
# 浏览器访问 → http://localhost:5000
```

然后在欢迎页点击「快速扫描」或直接输入仓库路径即可开始使用。
